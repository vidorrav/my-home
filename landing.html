AI CLI 工具配置指南

Claude Code Anthropic 官方 CLI

📋 系统要求

  • Windows 10/11 或 macOS 10.15+ 或 Linux
  • Node.js 18+ (LTS 版本推荐)
  • 网络连接
1

安装 Node.js

方法一：使用官方安装包（推荐）

  • 访问 https://nodejs.org
  • 下载 LTS 版本的安装包
  • 运行安装程序，按默认设置完成安装

方法二：使用包管理器

# Windows (使用 Winget) winget install OpenJS.NodeJS.LTS # macOS (使用 Homebrew) brew install node # 验证安装 node --version npm --version
💡 建议使用 LTS 版本以获得最佳稳定性。安装完成后需重启命令行窗口。
2

安装 Claude Code CLI

打开命令提示符（以管理员身份运行）或终端，执行以下命令：

npm install -g @anthropic-ai/claude-code

验证安装：

claude --version
⚠️ 如果遇到权限问题，请确保以管理员身份运行命令提示符。
3

配置 MHcoding API

3.1 获取 API Key

  • 登录 MHcoding 控制台
  • 点击「添加令牌」
  • 令牌名称：随意填写
  • 复制生成的 API Key

3.2 配置方式（推荐使用配置文件，永久生效）

创建配置文件：

# Windows 配置文件路径 %USERPROFILE%\.claude\settings.json # macOS/Linux 配置文件路径 ~/.claude/settings.json

配置文件内容：

{ "env": { "ANTHROPIC_AUTH_TOKEN": "你的API Key", "ANTHROPIC_BASE_URL": "https://你的域名" } }
💡 配置文件更加安全且便于管理，修改后需要重启 Claude Code 才生效。
4

启动 Claude Code

先进入到项目目录，然后启动：

cd your-project-folder claude

首次启动需要完成以下设置：

  • 选择喜欢的主题（回车）
  • 确认安全须知（回车）
  • 使用默认 Terminal 配置（回车）
  • 信任工作目录（回车）
  • 开始编程！🚀

Codex CLI OpenAI 官方 CLI

📋 系统要求

  • Windows 10/11 或 macOS 10.15+ 或 Linux
  • Node.js 22+ (推荐)
  • Git (可选，用于版本控制)
1

安装 Node.js

方法一：使用官方安装包（推荐）

  • 访问 https://nodejs.org
  • 下载 Node.js 22+ 版本的安装包
  • 运行安装程序，按默认设置完成安装

方法二：使用包管理器

# Windows (使用 Winget) winget install OpenJS.NodeJS.LTS # macOS (使用 Homebrew) brew install node # 验证安装 node --version npm --version
💡 Codex CLI 需要 Node.js 22+ 版本，请确保安装最新版本。
2

安装 Codex CLI

打开命令提示符（以管理员身份运行）或终端，执行以下命令：

npm install -g @openai/codex

验证安装：

codex --version
⚠️ 如果遇到权限问题，请确保以管理员身份运行命令提示符。
3

配置 MHcoding API

3.1 获取 Codex 专用 API Token

  • 登录 MHcoding 控制台
  • 注册账户或登录现有账户
  • 进入「API 密钥」页面
  • 点击「创建新密钥」，选择 Codex 专用分组
  • 复制生成的 API Key
⚠️ 重要：Codex 需要使用专门的分组令牌，与 Claude Code 的令牌不同！

3.2 创建配置文件夹

# Windows mkdir %USERPROFILE%\.codex cd %USERPROFILE%\.codex # macOS/Linux mkdir -p ~/.codex cd ~/.codex

3.3 创建配置文件

1. 创建 config.toml 文件：

model_provider = "mhcoding" model = "gpt-5.2-codex" model_reasoning_effort = "xhigh" network_access = "enabled" disable_response_storage = true [model_providers.mhcoding] name = "mhcoding" base_url = "https://你的域名/v1" wire_api = "responses" requires_openai_auth = true

2. 创建 auth.json 文件（在同一目录下）：

{ "OPENAI_API_KEY": "粘贴你的Codex专用分组令牌key" }
💡 配置完成后需要重启 Codex CLI 才生效。
4

启动 Codex CLI

先进入到项目目录，然后启动：

cd your-project-folder codex

首次启动需要完成以下设置：

  • 选择喜欢的主题（回车）
  • 确认安全须知（回车）
  • 信任工作目录（回车）
  • 开始编程！🚀

3

配置 MHcoding API

⚠️ 重要：请将下方的 GEMINI_API_KEY 替换为您在控制台生成的 Gemini CLI 专用 API 密钥！

3.1 创建 .gemini 文件夹

# Windows mkdir %USERPROFILE%\.gemini cd %USERPROFILE%\.gemini # macOS/Linux mkdir -p ~/.gemini cd ~/.gemini

3.2 创建 .env 文件

在 .gemini 文件夹中创建 .env 文件：

GOOGLE_GEMINI_BASE_URL=https://你的域名 GEMINI_API_KEY=粘贴你的Gemini CLI专用分组令牌key GEMINI_MODEL=gemini-3-pro

3.3 创建 settings.json 文件

在 .gemini 文件夹中创建 settings.json 文件：

{ "ide": { "enabled": true }, "security": { "auth": { "selectedType": "gemini-api-key" } } }
💡 配置文件更加安全且便于管理，需要重启 Gemini CLI 才生效。
4
