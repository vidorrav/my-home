C

Claude Code

偏“结对编程”体验：适合代码理解、重构、测试生成与复杂任务拆解。用 MHcoding API 统一转发与鉴权。

下一个：Codex

安装命令

选择平台 → 一键复制
Terminal 
# macOS (示例)
brew install claude-code
提示：如果你使用公司镜像源/代理，优先在终端里配置好 HTTP(S)_PROXY 或 npm registry。

MHcoding API 配置示例

将网关与 Key 注入 CLI
config.json 
{
  "provider": "mhcoding-api",
  "base_url": "https://YOUR_MHCODING_HOST/v1",
  "api_key_env": "NEWAPI_API_KEY",
  "model": "claude-4.5-sonnet",
  "headers": {
    "X-Client": "claude-code"
  }
}
长上下文与理解力 适合读大仓库、解释复杂模块与设计取舍。
重构与测试 把“改动→验证→回滚”做成可控流程。
风格一致性 配合规则/提示词，稳定输出团队风格。
网关统一鉴权 只维护一个 Key，按工具/模型路由。

Codex CLI

面向终端的高效编码助手：适合脚本生成、项目初始化、批量改动与可复现的命令式工作流。

下一个：Gemini

安装命令

平台选择 / 可复制
Terminal 
# macOS (示例)
brew install codex
建议：把常用指令封装成脚本/Makefile，让 AI 输出更容易落地与复现。

MHcoding API 配置示例

统一 endpoint + key
.env / config 
# 方式 A：环境变量（推荐）
export NEWAPI_API_KEY="YOUR_API_KEY"
export NEWAPI_BASE_URL="https://YOUR_MHCODING_HOST/v1"

# 方式 B：Codex CLI 参数（示例）
codex --base-url "$NEWAPI_BASE_URL" --api-key "$NEWAPI_API_KEY"
命令式工作流 适合脚手架、批量变更与自动化操作。
可复现性强 把提示词/命令固化进脚本，便于团队协作。
工具链融合 与 Git / CI / Lint / Test 组合，减少手工步骤。
统一网关路由 在 MHcoding API 内按模型/成本/权限策略分发。
G

Gemini CLI

偏多模态与工程效率：适合与文档、规范、代码生成联动。通过 MHcoding API 统一鉴权与可观测。

进入 Quick Start

安装命令

平台选择 / 可复制
Terminal 
# macOS (示例)
brew install gemini-cli
建议：把规范/接口文档放进提示上下文，输出更贴近真实工程约束。

MHcoding API 配置示例

示例为“网关转发”结构
config.json 
{
  "provider": "mhcoding-api",
  "base_url": "https://YOUR_MHCODING_HOST/v1",
  "api_key_env": "NEWAPI_API_KEY",
  "model": "gemini-2.5-pro",
  "headers": {
    "X-Client": "gemini-cli"
  }
}
文档驱动开发 从 PRD/接口/规范生成实现与测试。
多模态潜力 在需要时融合截图/图表/说明（按工具能力）。
长链路任务 适合“需求 → 方案 → 实现 → 验证”的流水线。
网关观测 统一记录调用来源、模型与成本（按你的网关能力）。
